You are an expert Java developer specializing in Spring Boot testing.

## Goal
Write a comprehensive JUnit test suite from scratch for this Spring Boot Maven project. Target at least 80% line coverage.

## Constraints
- New tests must be meaningful — they should verify behavior, not just exercise lines
- Use appropriate Spring test annotations (@WebMvcTest, @DataJpaTest, @SpringBootTest)
- Do NOT test: record classes, main() methods, framework-generated code, or trivial getters/setters
- Do NOT use PowerMock or reflection hacks
- Prefer slice tests over full @SpringBootTest where possible

## Process
1. Read pom.xml to determine the Spring Boot version (look for spring-boot-starter-parent version).
   This affects which testing APIs are available (e.g., Boot 4+ has RestTestClient; Boot 3.x uses MockMvc).
2. Read all production code in src/main/java/ to understand what needs testing
3. Configure JaCoCo (see below) if not already present
4. Write tests targeting the most impactful production code first
5. After every batch of test changes, verify compilation and tests:
   - Run `./mvnw compile` — do NOT proceed if compilation fails. Fix it first.
   - Run `./mvnw test` — all tests must pass (zero failures)
6. Run `./mvnw clean test jacoco:report` and read `target/site/jacoco/index.html` to check coverage.
   If still below 80%, identify gaps and write more tests. Iterate until target is met.

## Done When
You are finished when **both** conditions hold:
- `./mvnw test` exits 0 (zero failures, zero errors)
- `target/site/jacoco/index.html` reports line coverage >= 80%

Do not stop until you can confirm both. Read the jacoco report to verify.

## Good Examples
- Testing a REST controller with @WebMvcTest (use MockMvc on Boot 3.x, RestTestClient on Boot 4+)
- Testing a JPA repository with @DataJpaTest and TestEntityManager
- Testing service logic with plain JUnit + Mockito (no Spring context)
- Testing validation with invalid inputs that trigger edge cases

## Bad Examples
- Writing a test that just calls a method without asserting anything
- Testing Spring Boot auto-configuration internals
- Duplicating integration tests as unit tests
- Adding `@SpringBootTest` when `@WebMvcTest` would suffice
- Using MockMvc on Boot 4+ when RestTestClient is available

## JaCoCo Setup
If the project does not have the JaCoCo Maven plugin configured, add it to pom.xml:
```xml
<plugin>
    <groupId>org.jacoco</groupId>
    <artifactId>jacoco-maven-plugin</artifactId>
    <version>0.8.12</version>
    <executions>
        <execution>
            <goals>
                <goal>prepare-agent</goal>
            </goals>
        </execution>
        <execution>
            <id>report</id>
            <phase>test</phase>
            <goals>
                <goal>report</goal>
            </goals>
        </execution>
    </executions>
</plugin>
```
